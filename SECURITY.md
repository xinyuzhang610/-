# 安全策略

智教通涉及 API 密钥、数据库账号、学习记录和竞赛报名资料。请负责任地披露安全问题，不要在公开 Issue、Pull Request、讨论区或截图中发布漏洞细节、密钥或个人信息。

## 私下报告

优先通过仓库的 GitHub Security Advisory 页面选择 **Report a vulnerability** 提交私密报告：

<https://github.com/xinyuzhang610/-/security/advisories/new>

如果该入口暂时不可用，请联系仓库所有者或当前维护者，并仅说明需要建立私密沟通渠道；在安全渠道建立前不要发送利用代码、真实密钥或个人数据。

报告请包含：

- 受影响的页面、接口、版本或提交。
- 可重复的最小步骤和所需前置条件。
- 对账号、数据、可用性或隐私的实际影响。
- 已进行脱敏的日志、请求或截图。
- 建议的缓解方式（如有）。

## 响应原则

维护者会先确认问题是否可复现和影响范围，再协商修复、密钥轮换、数据通知与公开时间。修复发布前，请给维护者合理时间完成验证；未经双方确认不要公开漏洞细节。

## 敏感信息处置

- 发现真实密钥时应立即撤销并轮换，删除文件并不能使已进入 Git 历史的密钥失效。
- 发现报名表或学习记录泄露时应停止继续复制，记录最小必要证据，并通知有权限的维护者评估影响。
- 不要把生产数据导入演示环境；测试数据应使用虚构身份并去除联系方式。

一般功能缺陷、界面问题和不涉及安全的数据错误可以使用普通 Issue 模板报告。
