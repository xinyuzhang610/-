# 本地运行与验证指南

本项目最终交付版只要求在本地运行。代码入口为 `source/frontend` 和
`source/backend`，完整的七份比赛 Word 文档在 `docs/competition/final/`。

## 环境要求

- Node.js 20+
- Python 3.11+
- MySQL 8+
- Git

## 一次性准备

```powershell
cd source/backend
py -3.11 -m pip install -r requirements.txt

# 在 MySQL 中创建数据库；账号密码按本机实际情况填写
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS zjiaotong CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
Copy-Item .env.development.example .env
# 编辑 .env，至少修改 DATABASE_URL 和 SECRET_KEY
py -3.11 -m alembic -c alembic.ini upgrade head
py -3.11 seed_local_demo.py

cd ..\frontend
npm ci
Copy-Item .env.example .env
```

本地前端示例默认 `VITE_DEMO_MODE=true`，页面演示无需 Turnstile；后端示例使用 mock AI，不需要 DeepSeek Key。要联调真实登录和数据库功能时，将前端演示模式改为 `false`。

执行种子脚本后可用以下本地测试账号：`teacher_demo / Teacher2026`、`student_demo / Student2026`、`admin_demo / Admin2026`。账号只用于本机演示。

## 启动

终端一：

```powershell
cd source/backend
py -3.11 -m uvicorn main:app --host 127.0.0.1 --port 8000
```

终端二：

```powershell
cd source/frontend
npm run dev
```

访问 `http://127.0.0.1:3000`。后端检查地址为
`http://127.0.0.1:8000/api/health`，接口文档为
`http://127.0.0.1:8000/docs`。

## 快速验证

```powershell
cd source/frontend
npm test -- --run
npm run build

cd ..\backend
pytest -q
python -m compileall -q .
```

不要提交 `.env`、数据库密码、AI Key 或验证码密钥。最终运行只使用 `source/` 当前源码。
