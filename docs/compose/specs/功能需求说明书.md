## 2. 项目概述

### 2.1 项目定位

#### 2.1.1 一句话定义

> 一个面向教育场景的Web应用平台，通过**需求引导**帮助小白老师发现教学需求、体验AI智能体工具、并引导他们使用智能体平台创建自己的学习工具。

#### 2.1.2 核心价值（重点）

```
┌─────────────────────────────────────────────────────────────┐
│                    核心价值：需求引导                          │
│                                                             │
│  老师是小白 → 不知道自己要什么 → 需要引导                      │
│                                                             │
│  第1层：文理分科（缩小范围）                                   │
│      ↓                                                      │
│  第2层：需求引导（发现真实需求）                                │
│      ↓                                                      │
│  第3层：工具推荐/创建（解决问题）                               │
│      ↓                                                      │
│  第4层：数据看板（看到效果，形成闭环）                          │
│                                                             │
│  这才是评审会认可的价值：                                      │
│  ✅ 有引导流程（不是静态页面）                                 │
│  ✅ 有AI交互（DeepSeek API实际调用）                           │
│  ✅ 有数据存储（MySQL记录使用数据）                            │
│  ✅ 有业务逻辑（推荐算法、工具管理）                            │
│  ✅ 有前后端分离架构（Vue3 + FastAPI）                         │
└─────────────────────────────────────────────────────────────┘
```

#### 2.1.3 与师大平台的关系

```
师大平台 = 专业版（老师在平台上搭建复杂的工作流和Agent）
我们的Web应用 = 应用层（引导、体验、跳转）

两者不是替代关系，而是互补关系：
  我们的系统负责：引导需求 → 推荐工具 → 体验 → 跳转到平台
  师大平台负责：实际的Agent创建和运行

注意：师大平台接入先放一边，预留接口，后续接入
```

### 2.2 项目目标

#### 2.2.1 功能目标

1. **实现需求引导流程**：通过多步骤交互帮助教师发现和明确教学需求
2. **提供AI智能问答**：集成DeepSeek API，支持文本生成
3. **构建工具广场**：分类展示和提供各类AI智能体工具
4. **实现工具管理**：支持工具的创建、编辑、分享和使用统计
5. **建立数据看板**：提供使用数据的可视化展示和分析

#### 2.2.2 技术目标

1. **前后端分离架构**：采用Vue3 + Element Plus前端，Python FastAPI后端
2. **数据库支持**：使用MySQL 8.0存储所有业务数据
3. **AI能力集成**：集成DeepSeek API（模型：dsv4）实现文本生成
4. **可部署性**：支持Nginx + uvicorn部署，确保可运行

#### 2.2.3 竞赛目标

1. **满足比赛要求**：确保作品符合Web应用开发类的所有要求
2. **突出创新点**：重点展示需求引导流程和AI交互功能
3. **确保可演示**：保证所有功能都能正常运行，支持现场答辩演示
4. **文档完整**：提供完整的7份设计文档

### 2.3 系统边界

#### 2.3.1 系统包含的功能

- 教师端：登录注册、需求引导、工具管理、数据查看
- 学生端：工具使用、AI问答、工具广场、学习记录
- 管理员后台：工具管理、用户管理、系统配置（可选）

> **【待讨论】管理员后台是否实现？**
> - 方案A：完整实现（增加2-3天工作量）
> - 方案B：仅预留接口（推荐，时间紧张时可跳过）
> - 方案C：不实现
> - 详见：待讨论事项.md 第2.1节

#### 2.3.2 系统不包含的功能

- 不实现复杂的工具创建工作流（由师大平台负责）
- 不实现多模态内容生成（仅支持文本生成）
- 不实现复杂的推荐算法（使用简单的条件匹配）
- 不实现移动端适配（仅支持Web端）

## 3. 系统功能需求

### 3.1 教师端功能

#### 3.1.1 登录注册

**功能描述：** 教师用户可以通过账号密码登录系统，新用户可以注册账号。

**功能要求：**

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 账号密码登录 | 输入用户名和密码进行登录 | 高 |
| 用户注册 | 填写用户名、密码、姓名、学校、科目、年级等信息 | 高 |
| 密码找回 | 通过邮箱或手机号找回密码 | 中 |
| 记住登录状态 | 选择是否记住登录状态 | 低 |

**业务流程：**

```
用户打开登录页面 → 输入用户名和密码 → 点击登录
    ↓
系统验证用户名和密码 → 验证成功 → 进入教师端首页
    ↓
验证失败 → 提示错误信息 → 返回登录页面

新用户点击注册 → 填写注册信息 → 提交注册
    ↓
系统验证信息完整性 → 验证成功 → 自动登录并进入首页
    ↓
验证失败 → 提示错误信息 → 返回注册页面
```

## 7. 数据流设计

### 7.1 AI对话数据流

```
用户输入 → 前端发送请求 → 后端接收
    ↓
后端从数据库读取工具配置（提示词模板）
    ↓
拼接：系统提示词 + 用户输入
    ↓
调用DeepSeek API（model: dsv4）
    ↓
获取返回内容
    ↓
返回给前端展示

示例：
  系统提示词："你是一个古诗词赏析助手，请用趣味的方式..."
  用户输入："静夜思"
  拼接后发送给API → 返回赏析内容 → 前端渲染展示
```

### 7.2 工具推荐数据流

```
教师选择文理分科 → 后端获取选择结果
    ↓
教师选择需求类型 → 后端获取选择结果
    ↓
后端根据选择结果查询推荐规则表
    ↓
获取匹配的工具ID列表
    ↓
查询工具详情 → 返回推荐工具列表
    ↓
前端展示推荐工具
```
