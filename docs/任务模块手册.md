## M1: 数据库初始化与验证

### 模块目标
确保数据库表结构正确、种子数据完整、后端能正常连接数据库。

### 子任务

#### M1.1: 执行数据库初始化脚本
**任务描述**：执行 init.sql 创建数据库和表结构

**操作步骤**：
1. 登录MySQL：`mysql -u root -p`
2. 执行脚本：`source path/to/init.sql`
3. 验证表是否创建成功

**验收标准**：
- [ ] 数据库 `zjiaotong` 已创建
- [ ] 6张表已创建（users, tools, tool_templates, usage_logs, favorites, recommend_rules）
- [ ] 表结构与设计文档一致

**可修改空间**：
- 如需新增字段，可在对应表执行 `ALTER TABLE`
- 如需新增表，可编写新的SQL脚本

---

#### M1.2: 导入种子数据
**任务描述**：执行 seed.sql 导入预设工具模板

**操作步骤**：
1. 执行种子数据脚本：`source path/to/seed.sql`
2. 验证数据是否导入成功

**验收标准**：
- [ ] tool_templates 表有14条记录（6文科+5理科+3通用）
- [ ] 数据内容与设计文档一致

**可修改空间**：
- 新增工具模板：直接 INSERT INTO tool_templates
- 修改工具描述：UPDATE tool_templates SET description=...
- 删除工具模板：DELETE FROM tool_templates WHERE id=...

---

#### M1.3: 验证后端数据库连接
**任务描述**：确认后端能正常连接数据库

**操作步骤**：
1. 启动后端服务
2. 调用任意API接口
3. 检查后端日志无数据库连接错误

**验收标准**：
- [ ] 后端启动无报错

**可修改空间**：

#### M4.2: 学生端需求引导流程实现
**任务描述**：实现学生端完整的需求引导流程，帮助学生挖掘需求、分析需求、找到解决方案

**流程设计**：
1. **需求挖掘**：通过问答引导学生说出学习困惑
   - "你在学习中遇到什么困难？"
   - "哪门课让你觉得最难？"
   - "你希望解决什么问题？"

2. **需求分析**：帮助学生梳理问题类型
   - 知识理解类：概念不懂、原理不清
   - 解题方法类：不会做题、方法不对
   - 学习兴趣类：没兴趣、学不进去
   - 学习效率类：效率低、记不住

3. **需求匹配**：根据问题类型推荐合适的工具
   - 知识理解 → 名词解释器、概念辨析器
   - 解题方法 → 公式推导助手、错题分析器
   - 学习兴趣 → 古诗词趣味赏析、逻辑思维训练
   - 学习效率 → 知识卡片生成器、思维导图助手

4. **工具使用**：引导学生使用推荐工具解决问题

**验收标准**：
- [ ] 需求挖掘：有引导性问题，帮助学生表达困惑
- [ ] 需求分析：能识别问题类型并分类
- [ ] 需求匹配：根据问题类型推荐合适工具
- [ ] 引导完整：从困惑到解决方案的完整路径
- [ ] 可返回修改：支持返回上一步重新选择

**可修改空间**：
- 修改引导问题：修改 `views/student/Guidance.vue` 中的问题设计
- 修改问题分类：修改后端推荐逻辑
- 新增引导步骤：在前端添加新的步骤组件
- 修改推荐算法：修改 `services/recommend_service.py`
