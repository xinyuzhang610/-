# 首屏生命水墨显影设计

## 目标

将首屏现有的永久擦除改为跟随光标、自动闭合的生命水墨显影。用户移动鼠标时，只在当前轨迹附近短暂看见下层东方未来主义插画；光标移开后，旧区域在约半秒内恢复为完整墨色遮罩。

## 参考效果

参考 `https://mimo.xiaomi.com/coder` 的交互机制，但继续使用智教通现有视觉主题与生成插画。参考页的关键行为参数为：

- 墨点初始半径约 `8px`
- 墨点最大半径约 `128px`，带约 `45%` 随机变化
- 单个墨点生命周期约 `520ms`
- 沿鼠标轨迹每 `12px` 补一个墨点
- 同时存活墨点上限为 `160`
- 每帧先重铺完整遮罩，再使用 `destination-out` 挖出仍在生命周期内的墨点

## 根因

当前 `HeroScratchReveal` 只在移动时对 Canvas 执行永久擦除，并将所有标准化轨迹长期保存。组件缩放时还会重放历史轨迹，因此旧区域不会恢复。

此外，`.hero-scratch` 容器自身设置了实体深色背景。Canvas 透明后首先露出的是这个容器背景，而不是下层 `hero-ink-landscape.webp`，所以显影区域看起来仍是一条黑色轨迹。

## 设计

### 图层

首屏保持三层结构：

1. `hero__art`：现有生成插画、山水装饰和知识核心，位于底层。
2. `HeroScratchReveal`：透明容器中的 Canvas 墨色遮罩，位于中层。
3. `hero__content`：标题、说明和入口按钮，位于顶层。

遮罩容器不再绘制实体背景。Canvas 自身负责铺满墨色，因此被挖空的位置会直接显示下层插画。

### 动画模型

每个轨迹墨点保存坐标、出生时间、随机种子和最大半径。动画帧执行以下流程：

1. 使用东方深墨渐变重新铺满整个 Canvas。
2. 删除生命周期已超过 `520ms` 的墨点。
3. 将存活墨点从 `8px` 使用 `easeOutCubic` 扩张至随机最大半径。
4. 使用 `1 - t²` 逐渐降低挖空强度，使遮罩自然闭合。
5. 没有存活墨点后停止 `requestAnimationFrame`，避免空闲耗电。

墨点边缘使用 32 段不规则轮廓，并叠加三组不同频率的正弦扰动。中心保持清晰显影，外缘通过径向渐变形成水墨渗化。

### 轨迹与状态

- 桌面端在 `pointerenter` 和 `pointermove` 时生成墨点，无需按住鼠标。
- 长距离移动按 `12px` 间距插值，避免断裂。
- `pointerleave` 重置上一个坐标，但已生成墨点继续自然闭合。
- 不保存永久轨迹，不在 resize 后重放历史擦痕。
- resize 重新设置 DPR 和画布尺寸，并立即恢复完整遮罩。
- 首次产生有效显影后隐藏操作提示。

### 插画表现

继续使用已生成的 `hero-ink-landscape.webp`，不新增素材。图片恢复为更接近原图的亮度；左侧文字区保留局部深色渐变保证可读性，右侧知识核心和山水区域减少压暗，使短暂显影具有明显视觉奖励。

### 触屏与可访问性

- 仅在支持 hover 且未开启减少动态时启用生命水墨 Canvas。
- 触屏设备、低内存设备、Canvas 不可用和 `prefers-reduced-motion` 情况下进入静态模式，隐藏遮罩并直接展示完整插画。
- Canvas 保持 `aria-hidden="true"`，不阻挡键盘操作；内容链接和按钮继续位于顶层。
- 组件卸载时清理事件、观察器和动画帧。

## 测试与验收

自动化测试先失败再实现，至少覆盖：

- 动画帧会先重铺遮罩，再挖出存活墨点。
- 超过 `520ms` 的墨点被移除，旧区域恢复。
- 路径按固定间距插值，不产生长直线缺口。
- 离开再进入不会连接两个不相邻区域。
- resize 不重放历史轨迹。
- 减少动态和不可用环境进入静态模式。
- 卸载时停止动画并清理观察器。

浏览器验收覆盖桌面宽屏、`1088x778`、`851x778` 和移动端尺寸。重点确认下层插画可见、旧轨迹会闭合、内容可点击、没有横向溢出、控制台无错误。
