# 智教通前端重构设计规格

日期：2026-07-13
状态：已通过对话评审，待用户审阅书面规格
目标分支：`codex/eastern-future-ui-design`

## 1. 项目目标

在保留 Vue 3、现有路由结构和后端 API 业务含义的前提下，将智教通重构为适合竞赛大屏展示的高完成度产品。设计必须在用户打开首页的数秒内建立强烈记忆点，同时让教师和学生能够清晰完成真实任务。

主要成功标准：

- 首屏具有明确、原创且与教育场景相关的视觉记忆点。
- 教师端和学生端使用同一套品牌、组件和动效语言。
- 首页、身份选择、教师路径和学生路径均可完整演示。
- 1440p、1080p 大屏流畅运行，平板和手机不产生横向溢出。
- 视觉效果不掩盖信息层级、操作反馈与无障碍需求。
- 后端不可用时可通过明确标识的演示数据完成界面展示；正式模式仍调用真实 API。

## 2. 设计方向

采用“东方未来主义”，核心叙事为“知识觉醒”。视觉不是传统古风，也不是通用蓝紫科技模板，而是将墨色空间、鎏金印记、青绿能量、数字山水和知识星图转译为未来教育系统。

贯穿产品的核心视觉对象为“智核”：一个由文字粒子、知识节点和能量轨迹组成的抽象智能核心。智核出现在首页首屏、身份选择、功能宇宙和关键加载状态中，承担品牌记忆点与状态反馈双重作用。

### 2.1 色彩

- 墨夜：全站主背景，使用有层次的近黑色而非纯黑。
- 鎏金：教育、传承、重点操作和高价值信息。
- 青绿：智能、生长、连接和实时状态。
- 月白：正文、辅助界面与高对比信息。
- 朱砂：印章、重要警告和极少量强调状态。

颜色通过语义设计令牌集中管理，组件内不散落重复色值。

### 2.2 字体与图形

- 标题采用具有现代结构的中文宋体或美术黑体。
- 正文采用清晰的人文无衬线字体，并提供可靠的本地回退。
- Logo、导航图标、装饰线和图表符号使用代码原生 SVG。
- 不使用 emoji 作为功能图标，不使用机器人作为 AI 的默认形象。

## 3. 信息架构

### 3.1 营销首页

首页使用连续滚动叙事，而非相互割裂的功能卡片堆叠。

1. **知识觉醒**：动态数字山水、智核、巨型标题，以及教师/学生双入口。
2. **教学困境**：备课耗时、资源分散、学习差异等问题转化为动态图形场景。
3. **能力宇宙**：需求发现、智能推荐、AI 问答、工具广场、学习分析围绕智核运行。
4. **双端路径**：滚动时分裂为教师端与学生端两条光路，展示各自工作流。
5. **产品实景**：在动态设备框架中展示真实应用界面。
6. **最终召唤**：山水与知识星图汇聚，引导登录、注册或学生直接体验。

### 3.2 教师端

- 今日教学驾驶舱：课程状态、工具使用量、学生活跃度和 AI 建议。
- 需求引导：选择学科领域、明确教学目标、生成工具推荐。
- 我的工具：区分预设、自建和外部平台工具，支持现有管理操作。
- 数据洞察：趋势、热门工具、学生反馈和最近使用记录。

需求引导中的知识星图随步骤逐渐完成，用空间连续性帮助用户理解进度。

### 3.3 学生端

- 兴趣引导：保留“戳泡泡”创意，升级为可探索的知识宇宙。
- 工具广场：文科、理科、通用分类，以及搜索与热门推荐。
- AI 对话：左侧学习上下文与工具信息，右侧为对话主区域。
- 学习轨迹：展示统计数字、使用记录和连续学习状态。

### 3.4 登录与身份

- 首次进入允许选择教师或学生身份。
- 教师进入登录/注册流程。
- 学生可直接探索，也可通过分享链接进入指定工具。
- 身份改变背景场景和后续导航，但不形成两个割裂的品牌系统。

## 4. 动效系统

### 4.1 首次进入

墨滴扩散形成山脉，文字粒子汇入智核，标题分层显现。主动画约 2.5 秒，可跳过，不阻塞主要操作。

### 4.2 输入响应

- 智核、光线和雾层对鼠标产生轻度视差。
- 按钮附近粒子产生克制的吸引反馈。
- 不使用持续追踪光标或妨碍阅读的强闪烁效果。

### 4.3 滚动叙事

章节之间使用语义变形：山水转换为数据曲线，书页转换为工具卡片，星轨分裂为双端路径。滚动触发逻辑必须支持快速滚动、回滚和窗口尺寸变化。

### 4.4 工作台动效

工作台内显著收敛动效。卡片使用材质、边缘光、扫描线和微小位移表达层级；AI 回复加载表现为知识节点组织文字；图表可使用山脉与星轨隐喻，但数值和坐标始终优先。

### 4.5 无障碍与降级

- 支持 `prefers-reduced-motion`。
- 降级模式关闭粒子、视差和连续变形，但保留完整内容与视觉层次。
- 键盘焦点始终可见，交互目标不小于 44×44 像素。
- 正文对比度目标不低于 WCAG AA。

## 5. 素材策略

使用图片生成能力制作项目专属素材：

- 超宽数字山水首页主视觉。
- 教师与学生抽象人物场景。
- 知识智核、教学工具宇宙和 AI 对话空间。

生成素材不包含文字、Logo 或水印，避免文字错误并方便响应式排版。最终素材保存到项目资源目录，转换为适合浏览器的 WebP/AVIF，并设置尺寸占位与延迟加载以控制 CLS。装饰性图像使用空 `alt`，承载内容的图像提供准确替代文本。

## 6. 前端架构

保留现有 Vue 3、Vue Router、Pinia、Element Plus 和 API 封装。重构集中于以下边界：

- `components/brand/`：智核、品牌标记、数字山水与装饰图形。
- `components/motion/`：滚动揭示、视差、粒子层和降级逻辑。
- `components/ui/`：按钮、卡片、状态提示、筛选和表单基础组件。
- `components/data/`：统计卡、趋势图、排行和记录列表。
- `layouts/`：营销首页、教师工作台和学生工作台布局。
- `styles/`：设计令牌、字体、全局材质与响应式规则。
- `composables/`：动效偏好、视口状态、演示模式和异步请求状态。

页面只组合组件与调用 API，不直接实现大段粒子或滚动引擎。高成本视觉模块按路由或视口延迟加载。

## 7. 数据流与演示模式

- 页面继续通过现有 `src/api/` 模块请求后端。
- Pinia 管理身份、登录状态和跨页面共享状态。
- 异步页面统一处理加载、成功、空数据、网络失败、AI 超时和未登录状态。
- 演示模式通过环境变量显式开启，并在界面显示“演示数据”标识。
- 演示模式不伪装为真实接口成功；正式模式不静默回退到演示数据。

## 8. 错误处理

- 网络失败：保留当前页面上下文，提供重试操作和可理解的错误信息。
- AI 超时：保留用户输入，允许重新生成，不插入伪造回复。
- 鉴权失效：提示原因后进入登录页，并保留可恢复的目标地址。
- 图像或高成本视觉加载失败：显示基于 CSS/SVG 的品牌背景，不出现空白区域。
- 不支持 WebGL 或低性能环境：自动使用静态智核和轻量 CSS 动效。

## 9. 性能目标

- 主要展示环境：高性能桌面电脑，1440p 与 1080p。
- 首屏关键文字与入口不依赖大型图片下载后才出现。
- 避免单个同步脚本阻塞主线程；非关键动效与图片延迟加载。
- 粒子数量根据设备像素比和帧率动态降级。
- 构建阶段检查包体，优先对大型视觉模块进行路由级拆分。

## 10. 验证计划

### 自动验证

- 现有 Vitest 测试保持通过。
- 为身份切换、核心导航、演示模式和异步状态补充测试。
- `npm run build` 成功，无未处理的编译错误。

### 浏览器验证

- 首页首屏加载、滚动叙事和所有行动入口。
- 教师：登录 → 需求引导 → 工具推荐 → 工具管理 → 数据看板。
- 学生：兴趣引导 → 工具广场 → AI 对话/工具使用 → 学习轨迹。
- 1440p、1080p、平板和手机视口。
- 键盘导航、减少动态效果、控制台错误和失败状态。

## 11. 实施边界

本轮负责前端视觉、交互、组件重构、素材接入和现有 API 对接。不修改数据库业务规则，不重写后端接口，不虚构后端尚未实现的功能。若现有接口无法支持某一已存在页面，应记录为集成缺口，并使用明确标识的演示状态完成视觉验收。
