# 智教通首页视觉反馈精修设计

**日期：** 2026-07-14

**状态：** 已完成交互式方案确认，等待书面规格复核

**视觉方向：** B「东方编辑式引路」

**擦显方向：** A「Canvas 柔边墨刷」

## 1. 目标

在不改变智教通教师端、学生端业务入口和既有页面叙事的前提下，解决首页评审中指出的八项视觉问题：

1. 首屏教师入口的字体与文字箭头突兀。
2. 首屏学生入口存在相同问题。
3. 问题区标题字体、断行和版式不稳定。
4. 教师端路径卡片在不同浏览器尺寸下裁切人物。
5. 学生端路径卡片存在相同问题。
6. 底部教师 CTA 的字体与文字箭头突兀。
7. 底部学生 CTA 存在相同问题。
8. 首屏缺少用户期望的鼠标移动擦除墨幕、逐步显现背景图的交互。

成功标准是：视觉语言统一、人物主体在目标视口完整、标题不产生机械碎行、入口可读可点、墨幕交互自然且有可靠降级。

## 2. 范围

### 范围内

- `HeroSection` 的入口按钮和墨幕擦显交互。
- `PainSection` 的标题文案层级、断行和响应式布局。
- `DualJourney` 的教师/学生人物画幅及卡片高度策略。
- `FinalCall` 的教师/学生 CTA。
- 支撑上述改动的共享小组件、自动测试和多视口浏览器验收。

### 范围外

- 教师端、学生端业务页面的功能或数据改造。
- 路由、权限、后端 API、数据库和业务流程调整。
- 替换现有教师/学生人物插画或首屏背景素材。
- 新增与八项反馈无关的首页模块。

## 3. 视觉设计

### 3.1 统一入口组件

首屏和底部 CTA 共用一个入口组件，使用相同的排版、间距、边框和悬停逻辑。

- 中文文字使用现有 UI 正文字体栈，中等字重，避免粗重衬线字体。
- 使用舒展但克制的字距，保证“教师入口”“学生入口”等短标签具有精致感。
- 删除裸字符 `→`，改用自绘细线 SVG「引路符」。
- 引路符悬停时沿水平方向移动约 4px；按钮本体不弹跳。
- 按钮采用轻微圆角与双层细框，不使用普通大圆角胶囊。
- 教师入口采用深墨底与暖金线条；学生入口采用深玉色底与浅金线条。
- 组件保留清晰的键盘焦点状态和足够的触控面积。

### 3.2 问题区标题

标题改为两个明确的语义行，不依赖浏览器自动断行：

> 问题不在工具多少<br>
> 而在知识尚未成路

第一行较克制，第二行承担视觉重音。标题容器设置理想行宽；中等视口提前调整网格比例或切换上下布局，避免文字列被挤压。正文与问题卡片维持既有信息，不做业务扩写。

### 3.3 教师/学生人物画幅

路径卡片取消固定 `48rem` 高度和单层 `object-fit: cover` 背景，改为四层画幅：

1. 同图放大、低饱和、柔化的氛围背景层；
2. 使用 `object-fit: contain` 的完整人物主体层；
3. 保证文字可读的底部渐变层；
4. 路径名称、说明与步骤内容层。

主体层以底部居中为主要锚点，不裁切头部、手臂或主要教学/学习姿态。卡片高度由画幅比例与内容决定，不再用固定超高值。桌面使用双列；空间不足时调整内部比例，手机切为单列。

## 4. 首屏墨幕擦显

### 4.1 图层关系

首屏背景图位于底层；Canvas 墨幕覆盖背景图；标题、说明、入口按钮和跳过控制位于墨幕上方。墨幕不会遮挡文字，也不会阻止入口点击。

### 4.2 输入行为

- 桌面端：指针在首屏移动即可擦除，无需按键。
- 触屏端：仅在手指按下并进行横向或斜向滑动时擦除；画布保留 `pan-y`，以纵向为主的手势继续滚动页面。
- 未擦除区域始终保留；不根据擦除比例自动清空或恢复。
- 首次擦动后，轻量提示“移动光标，唤醒画卷”淡出。

### 4.3 笔触算法

擦显组件对连续指针坐标做距离插值，避免快速移动产生断点。每个采样点使用柔边笔触，并按移动速度调整半径：快速移动略宽、边缘更舒展，慢速移动更细、更集中。少量确定范围内的纹理扰动用于弱化规则圆刷感，不使用持续高负载的粒子模拟。

Canvas 按设备像素比绘制。组件保存归一化轨迹；容器尺寸变化后重新建立画布并按新尺寸回放轨迹，因此已经擦过的部分不会突然复原。

### 4.4 跳过和降级

- 现有“跳过动态”同时清除墨幕，并停止后续擦显采样。
- `prefers-reduced-motion: reduce` 下直接展示静态背景。
- Canvas 上下文不可用、初始化失败，或浏览器明确报告 `deviceMemory <= 2` 时直接展示静态背景；没有 `deviceMemory` 信息时不凭设备类型猜测性能。
- 图片加载失败时保持可读的底色、文案和入口，不显示破损素材图标。
- 组件卸载时移除指针、触摸和尺寸监听，取消动画帧并释放轨迹数据。

## 5. 组件边界与数据流

### `HeroScratchReveal`

只负责 Canvas 初始化、容器尺寸、指针采样、轨迹回放、跳过和资源清理。它通过属性接收是否跳过，通过事件通知首次擦动；不包含首页文案和路由逻辑。

### `EntryLink`

通过属性接收角色、标签和目标链接，内部统一渲染文字、双层边框和 SVG 引路符。`HeroSection` 与 `FinalCall` 只负责传入内容。

### 路径人物画幅

`DualJourney` 中建立可复用的主体画幅结构，接收图片、对齐方式和替代文本。教师/学生卡片共享图层结构，允许颜色和文案不同。

### 标题语义行

`PainSection` 在模板中显式渲染两行标题。CSS 只负责字号与布局，不通过伪元素复制关键文案。

## 6. 可访问性与性能

- 入口保持语义链接，不改成不可键盘访问的容器点击。
- SVG 仅作装饰并从辅助技术中隐藏；链接文字提供完整名称。
- Canvas 为装饰性图层，不承担信息表达。
- 触控目标不小于 44px，高对比焦点环在浅色和深色背景上均清晰。
- 指针采样通过动画帧合并，避免每个原始事件立即重绘。
- 画布尺寸设置合理上限，防止高像素比大屏产生过大内存占用。

## 7. 测试与验收

### 自动测试

- 首屏和底部入口不存在裸字符箭头，均渲染 SVG 引路符。
- 教师/学生入口使用相同共享结构，并保留正确目标链接。
- 首屏存在擦显 Canvas；跳过或减少动态时进入静态状态。
- 擦显组件在卸载时完成监听器和动画帧清理。
- 路径卡片存在氛围背景层和 `contain` 主体层。
- 问题区标题包含两个稳定的语义行。
