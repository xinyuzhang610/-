## 一、项目核心创新点（必须牢记）

### 创新点定位

**不是"做一个AI教学平台"，而是"为零基础小白教会使用AI工具并挖掘兴趣"。**

### 创新点体现

| 创新点 | 在哪里体现 | 评委关注点 |
|--------|------------|------------|
| **需求引导流程** | 教师端3步引导 + 学生端3步引导 | 引导流程是否多步骤、有分支、有条件判断 |
| **AI实际调用** | DeepSeek API真实调用 | 是否真的调用了AI，不是假的 |
| **数据真实存储** | MySQL数据库存储 | 数据是否真正入库，不是写死的 |
| **流程闭环** | 注册→引导→使用→记录→看板 | 整个流程是否能走通 |

### 核心原则

1. **引导流程必须多步骤** — 不能一步到位，要有"文理分科→需求选择→工具推荐"的完整流程
2. **AI必须真实调用** — 不能用mock数据，必须调用DeepSeek API
3. **数据必须入库** — 用户注册、工具使用、推荐记录都要存到MySQL
4. **推荐用if-else** — 不用AI做推荐，用简单的条件匹配（评委能看懂逻辑）

### 5.3 任务清单

| 编号 | 任务 | 具体内容 | 做法 | 验收标准 |
|------|------|----------|------|----------|
| 2.1 | 替换Logo | 把临时SVG换成正式PNG | 用AI生成或Canva设计 | Logo清晰美观，尺寸合适 |
| 2.2 | 替换背景图 | hero-bg.png换成教学场景 | AI生成/Midjourney/免费图库 | 图片清晰，风格匹配 |
| 2.3 | 替换卡片图 | 5张特性卡片图片 | AI生成/免费图库 | 5张图都清晰美观 |
| 2.4 | 添加跳转 | "立即体验"按钮跳转到功能页 | 在HTML里加链接 | 点击按钮能跳转 |

### 5.4 验收方式

1. 双击 `index.html` 打开页面
2. 检查Logo是否清晰
3. 检查背景图是否美观
4. 检查5张卡片图是否都显示正常
5. 点击"立即体验"按钮，看是否能跳转

### 5.5 多方案对比

**方案A：用AI生成图片**
- 优点：快速、风格统一、免费
- 缺点：可能需要多次调整提示词
- 工具：Midjourney、DALL-E、文心一格

**方案B：用免费图库**
- 优点：图片质量高、真实感强
- 缺点：风格可能不统一、需要找合适的图
- 网站：Freepik、Pexels、Unsplash

**方案C：自己设计**
- 优点：完全可控、独一无二
- 缺点：耗时、需要设计能力
- 工具：Figma、Canva

**建议：方案A + 方案B 结合，用AI生成主体，用图库补充细节。**

## 六、模块三：前端功能页面（核心，你们做）

### 6.1 这是什么

项目的**核心功能页面**，用Vue3 + Element Plus开发的Web应用。包括登录注册、教师端、学生端的所有功能页面。

### 6.2 技术栈

| 技术 | 用途 | 说明 |
|------|------|------|
| Vue 3 | 前端框架 | 使用Composition API |
| Element Plus | UI组件库 | 提供按钮、表单、卡片等组件 |
| Axios | HTTP请求 | 调用后端API |
| Vue Router | 路由管理 | 页面跳转 |
| Pinia | 状态管理 | 存储用户信息等 |

### 6.3 模板选择

**推荐方案：基于vue-pure-admin模板改造**

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **vue-pure-admin** | 功能完整、文档好、社区活跃 | 需要删除不需要的模块 | ⭐⭐⭐⭐⭐ |
| vue-vben-admin | 功能强大、设计精美 | 太重、学习成本高 | ⭐⭐⭐ |
| vue-admin-better | 简单易用 | 功能较少 | ⭐⭐⭐ |
| 从零搭建 | 完全可控 | 耗时、容易出问题 | ⭐⭐ |

**建议：用vue-pure-admin，删掉不需要的模块，保留登录、布局、路由等功能。**

### 12.2 师大智能体平台接入方案

#### 平台简介

师大智能体平台是一个可以让用户**自己创建AI工具**的平台。用户可以：
- 定义工具名称和描述
- 编写提示词模板
- 设置参数
- 生成可使用的工具

#### 接入方案对比

| 方案 | 实现方式 | 优点 | 缺点 | 推荐度 |
|------|----------|------|------|--------|
| **方案A：跳转链接** | 点击按钮跳转到师大平台 | 实现最简单、快速上线 | 用户体验割裂 | ⭐⭐⭐⭐ |

| **方案C：API对接** | 调用师大平台API | 体验最好、完全集成 | 开发量大、需要API | ⭐⭐ |

#### 推荐方案：方案A（跳转链接）

**理由：**
1. 实现最简单，不影响主流程
2. 可以快速上线
3. 以后有需要再升级

**实现方式：**

在工具推荐页面（Step 6）增加一个按钮：

```html
<div class="create-tool-option">

  <a href="https://师大平台网址" target="_blank">
    <button>创建自定义工具</button>
  </a>
</div>
```
